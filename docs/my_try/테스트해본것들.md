## 🧪 직접 테스트한 방법 (2가지)

### 1️⃣ servers.json으로 실제 서버 테스트

**설정**: `servers.json`에 공개 Minecraft 서버 등록

```json
{
  "servers": [
    { "id": "mc-test-1", "name": "Hypixel", "type": "minecraft", "host": "mc.hypixel.net", "port": 25565 },
    { "id": "mc-test-2", "name": "CubeCraft", "type": "minecraft", "host": "play.cubecraft.net", "port": 25565 }
  ]
}
```

**테스트한 것**:
- `game_server_query` – gamedig로 서버 상태 조회 (이름, 맵, 인원, ping)
- `server_overview` – 등록된 서버 전체 조회
- `server_health`, `server_compare` 등

**특징**: RCON 없음 (query만). 실제 공개 서버에 접속해서 동작 확인.

---

### 2️⃣ 로컬 RCON mock 서버로 테스트

**설정**: Rust/CS2 없이 로컬에서 RCON 프로토콜만 구현한 테스트 서버 사용

| 단계 | 명령 |
|------|------|
| 터미널1 | `npm run rcon:server` → 127.0.0.1:27017에서 mock 서버 실행 |
| 터미널2 | `STEAM_MCP_SERVERS_PATH="./test/rcon/servers.rcon-test.json" SERVER_RCON_PASSWORD="test123" npm run rcon:test -- my-rust status` |

**테스트한 것**:
- `sendRcon` – RCON 클라이언트가 mock 서버에 연결해 `status` 명령 실행
- 응답: `hostname: RCON Test Server`, `players: 0`, `map: test`

**특징**: 실제 게임 서버 없이 RCON 연결·명령·응답 흐름 검증.
